Hosting personal domain website on github.

Creating and Hosting custom domain on github pages.

Posted by Akhil Gupta on March 20, 2016

Having a website with personal domain is necessary in today's world especially for tech people who want to reach to large number of audience.

Yeah!! For people like us Github has something known as Github Pages for hosting static website.

As a Github user, you're entitled to one free "user website" which will be live at http://yourusername.github.io. For this you have to create a repo yourusername.github.io and push all your website code in this repo.

You can also setup a custom domain on github pages.

Let's start!

First, you need to buy a custom domain from domain selling website, you can check out Godaddy,Hostgator,Hostinger and many more are available in market

Now some coding part, you can take a free templates for your website by googling a bit on internet and then make some changes in the source code according to your need or can start from the scratch.

Hosting

First, you have to create a repo with yourusername.github.io and push your website code in it, and you can see your webiste live on http://yourusername.github.io.

Add a CNAME file to your repo

For hosting your custom domain which you bought you have to create a file CNAME (in caps) and mention your domain name in it and commit it into your repo.

yourcutsomdomain.com

Domain setting

Now log in to website where you bought the domain, navigate to domain list and click on the domain which you want to host on github.
Click on manage DNS

You will see something like this
You need to configre two records for setting DNS

  1. A record
  2. CName

For A record, as shown above you have to enter @ for Host and 192.30.252.153 for ip address, you can set TTL to least time or automatic.

For CName , as shown below, set www for Host and yourusername.github.io for ip address, you can set TTL to least time or automatic.

For few domain sellers,for setting DNS you have to configure/set three records two A record and one CName. Configuration of CName is same as explained above, difference comes only in configuring ip address for A record for first A record you have to set ip address as 192.30.252.153 and for second A record you have to set ip address as 192.30.252.154. While setting for Host as @ remains same for both.
